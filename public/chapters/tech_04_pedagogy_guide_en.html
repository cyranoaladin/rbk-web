Trainer Guide

Conduct framework for trainers and mentors: posture, AI tooling usage, and technical grading criteria.

Mentoring vs Lecture

What we favor

  • Live coaching on real cases, pair-debugging, targeted reviews.
  • Guided questioning instead of giving answers to preserve autonomy.
  • Clear Definition of Done (tests, docs, perf, security).

To avoid

  • Long slide decks without practice.
  • Unreproducible fixes or missing tests.
  • Ignoring accessibility (UI), security (backend), or traceability.

Using AI tools in class

Preparation

Shared starter prompt, expected examples, explicit limits (no copy of critical code without review).

Execution

Compare 2-3 AI proposals, justify choices, validate via automated tests.

Traceability

Mention assistants in PRs, keep key prompts in sprint documentation.

Technical grading criteria

Axis Expectations Check
Code & Tests Unit + negative tests, invariant coverage, green CI. OK / To improve
Security Short threat model, applied security checklist, no hardcoded secrets. OK / To improve
Observability Structured logs, key metrics, documented rollback plan. OK / To improve
UX & Accessibility Keyboard nav, ARIA roles, handled and displayed errors. OK / To improve
