Syllabus Technique & Standards

La "Bible" technique. Structure de repo obligatoire, critères d'évaluation et plan opérationnel (S01-S48).

Structure de Dépôt (Standard)

  • repo-root/
  • README.md // Quickstart
  • RUNBOOK.md // How-to run/debug
  • PROOFS.md // Ledger hebdo
  • THREAT_MODEL.md // Risques
  • adr/ // Architecture Decisions
  • scripts/ // Automation (OBLIGATOIRE)
  • setup.sh
  • test.sh
  • build.sh

KO-Structure (Éliminatoire)

L'absence de PROOFS.md, RUNBOOK.md ou du dossier adr/ est un échec automatique. Aucune évaluation technique si non-conforme.

KO-Repro

Si ./scripts/setup.sh && ./scripts/test.sh échoue sur la machine de l'évaluateur, le projet est NO-GO.

Grille d'Évaluation (/100)

Critère Pts Niveaux de Performance
1. Fonctionnel 25 21-25: Conforme + Cas limites + Démo fluide.
2. Qualité Code 25 21-25: Tests Unit/Integ solides. Refactoring propre.
3. Sécurité 20 16-20: Threat Model complet + Liste de Contrôle + Preuves.
4. Ops & Obs. 15 12-15: Logs structurés, Métriques, Playbook.
5. Documentation 10 8-10: Repro en 1 commande. Vars d'env claires.
6. Discipline 5 5: PROOFS complet + ADR + Changelog.
