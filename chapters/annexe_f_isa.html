Annexe F — Bourse d'Excellence & ISA

Un mécanisme d'élite pour l'élite. L'ISA n'est plus un standard, mais une récompense. Réservé aux 2 meilleurs talents.

Principes & Éligibilité

Philosophie

L'étudiant ne paie que s'il dépasse un seuil. Le risque de crédit est porté par RBK ; la valeur créée est capturée à la source via capture programmable et oracle on-chain. Musharaka compatible.

Répartition des risques
  • RBK : financement, gouvernance & slashing
  • Nexus : exécution pédagogique & certification
  • Alumni Council : arbitrage décentralisé (Realms)

Critères d'Éligibilité Stricts

  • Top 2 Technique : Sélection sur classement Piscine (Algo + Soft Skills).
  • Quota Absolu : Limité à 2 bénéficiaires par cohorte, sans exception.
  • Objectif : Méritocratie absolue. Inclusion des hauts potentiels sans ressources.

Architecture Trustless (EVM)

Wallet & Abstraction

Safe 2/3 (Apprenant / Guardian RBK / Recovery). ERC-4337 + Paymaster sponsorise gaz de graduation & transferts initiaux. Zodiac Roles restreint destinations (pas de mixers, whiteliste RBK).

In-Share 90/10

0xSplits V2 immuable : 90% Trésor RBK → governance pool, 10% Pocket → wallet apprenant. Gelato/Chainlink automatise distribute() chaque semaine. Interdiction : mixers, routes non-KYC.

On-ramp & Oracles

Monerium IBAN → EURe on-chain. Request Network : factures routées directement Splitter. Reclaim zkTLS : preuves bancaires TND off-chain, SBT de revenus on-chain.

Engagement programmable

ERC-3525 (CPPS) minté à graduation : units=cap (20k TND), slots=matricule apprenant. amortize() décrémente value chaque mois. burn() auto à cap atteint ou 36 mois. Transférable via slot splitting.

Streaming 15%

Sablier V2 LockupDynamic : 15% de chaque revenu streamés seconde par seconde vers Trésor RBK. Cliff=0, duration=36 mois. Cancelable par apprenant (déclencheur d'alerte). Withdrawal auto hook.

Réputation & Arbitrage

SBT ERC-5192 (non-transférable, émetteur=RBK) : états Vert (OK) → Jaune (Alert) → Rouge (Défaut). Kleros Governor ERC-792 : verdict arbitrage → execution via proxy pour revoke/burn. Frais remboursés si apprenant gagnant.

Architecture Haute Fréquence (Solana)

Multisig OpSec

Squads V4 2/3 (Apprenant / Guardian RBK / Recovery). RBAC : Guardian peut proposer, RBK exécute. Spending limits : 10% SOL/USDC libre, transferts >threshold co-sign RBK. Sub-comptes pour segmenter fonds ($10B AUM sécurisé).

Split natif Pull

Metaplex Hydra Fanout : wallet de "fanout" reçoit USDC/SOL. PDA-based : chaque bénéficiaire a slot. Crank (Helius/Clockwork/Tuktuk) exécute distribute() → calcule (Total * % Part) - Réclamé. 90/10 split atomique.

On-ramp & Facturation

Kast : IBAN/ACH → USDC <1min. Helio Pay Streams : paiements récurrents pré-autorisés, Shopify webhook. Reclaim zkTLS : preuves revenu TND tunisien sans doxxing identité.

Engagement Token-2022

Token-2022 Mint non-transférable (via extension). Transfer Hook : rejette ANY transfert sauf burn (RBK delegate). Metadata slot : cap restant en UI. Permanent Delegate : RBK peut burn si défaut sans signature apprenant.

Streaming continu

Streamflow native Solana : flux seconde par seconde (15%), <0.0001 SOL frais. Withdrawable automatically via crank. Cancellable par apprenant → Watchtower alert. Pont EVM : Wormhole pour sync Sablier.

Gouvernance & DAO

SPL Governance (Realms) : vote communauté sur paramètres ISA (seuil, cap, durée). Alumni Council via Squads V4 : arbitrage premier niveau sur disputes. Verified builds : code on-chain matché GitHub source (Ellipsis). Optionnel : pont Kleros via Wormhole.

Standards d'Actifs Programmables (Token-2022)

Transfer Hooks

Logique programmée au protocole : chaque transfert invoque hook programme (RBK Governor). Cas d'usage ISA :
1. Vérifier status SBT apprenant (pas Rouge)
2. Rejeter si revenu < seuil (oracle check)
3. Logs audit immuable
Sécurité : comptes READ-ONLY durant hook, apprenant ne peut signer, hook ne peut drain.

Non-Transferable (Debt SBT)

Extension native : token intransférable dès mint. Seul Permanent Delegate (RBK) peut brûler. Use Case :
• Diplôme : non-transférable, autorité RBK revoke si fraude
• Tokens d'engagement : apprenant ne peut vendre, RBK gère burn
Récupération : delegate permet transfert vers new wallet si compromission (clé perdue).

Interest-Bearing Tokens

Séparation Amount UI vs Raw Amount : token affiche 100 mais raw=110 (10% rendement). Wallets calculent auto UI. Solana ISA : Debt tokens portent intérêt négatif (amortize) : 20k TND initial → décrémente vers 0 au fil des paiements, sans tx supplémentaires.

Confidential Transfers

zkTLS + Confidential Amounts : montants encrypted, adresses publiques. ISA Use Case : salaires apprenant vs RBK confidentiels (B2B), audit key (RBK) décrypte pour conformité. Limite MTU : tx Solana=1232 octets, Address Lookup Tables (ALTs) compressent références.

Identité Souveraine & Crédit Sous-Collatéralisé

Reclaim Protocol (zkTLS)

Zero-Knowledge TLS : apprenant génère preuve ZK qu'il s'est connecté à banque tunisienne HTTPS, données revenus ∈ [Min, Max]. Flow :
1. dApp demande "Revenu > 2500 TND?"
2. Client ouvre session HTTPS cachée (ADP, Tunisie banque)
3. MPC + TLS notarisation
4. Preuve ZK générée (sans doxxer creds)
5. Submit Solana/EVM → SBT "HighEarner" minté
Privacy : identité off-chain reste secrète, on-chain vérifiable.

Solana ID & Scoring

Solana Attestation Service (SAS) : agrège Reclaim + on-chain history → "Solana ID" unique. Score crédit :
• Historique remboursement (past ISA, loans)
• Solvabilité off-chain (via Reclaim)
• Présence réseau (staking, governance)
RociFi : prêts sous-collatéralisés via score. ISA Implication : apprenant avec score >700 peut accélérer ISA (délai réduit) ou receiving bonus (10% cap up).

Flux end-to-end ISA

1. Onboarding

EVM : Safe 2/3 déployé, ERC-4337 Paymaster activé, Zodiac Roles configured.

Solana : Squads V4 2/3 + spending limits, ALTs pré-chargées.

Both : SBT admission minté (Vert), Reclaim session initialisée, Splitter/Fanout déployés.

2. Formation (90/10)

Bounties (Genesis → Architecte) versées sur Splitter/Fanout.

90% auto → RBK Trésor + governance pool.

10% auto → Wallet apprenant (pocket money). Venture Engine vérifie routing chaque mois.

3. Graduation

EVM : ERC-3525 CPPS minté (20k TND cap, slot=matricule), SBT diplôme ERC-5192 actif.

Solana : Token-2022 non-transférable minté (transfer hooks, delegate RBK).

Sablier + Streamflow configurés (36 mois, 15%, 0 cliff).

4. Emploi & Détection

Invoice Oracle : Monerium (IBAN), Kast (ACH), Request Network (B2B), ou Reclaim zkTLS (paie).

Watchtower (Gelato/Chainlink) scrute revenu ∈ [2.5k, ∞) TND. Divergence → SBT Jaune (Challenge 10j).

Pause auto si revenu < 2.5k (pas de stream, pas de penalty).

5. Remboursement

EVM : Sablier stream 15% → RBK, ERC-3525 amortize() décrémente value.

Solana : Streamflow 15% → Fanout Trésor, Token-2022 amount réduit chaque retrait.

EURC bridge : convertit USDC → EURC → TND via DEX à taux marché (Orca/Aerodrome).

6. Sorties

Normal : Cap 20k ou 36 mois atteint → ERC-3525/Token-2022 burn() auto, SBT reste Vert.

Défaut : SBT Rouge → Kleros (EVM) ou Alumni Council (Solana) arbitrage. Si verdict RBK, burn exécuté.

Profil public : Arcx/Spectra met à jour score ISA (repos building possible après 2 ans).

Contrôles, Slashing & Arbitrage Avancés

Surveillance (Watchtower)

EVM : Gelato AutomationV2 scrute Monerium IBAN → balance. Divergence (revenu off-chain vs stream) → alerter.

Solana : Clockwork crank sur Reclaim proof. Si preuve expirée ou revenu

Challenge : 10 jours pour apprenant justifier (nouvelle preuve Reclaim, paie attestée). Silence → Rouge.

Réputation & Scoring

SBT Multi-État : Vert (OK) → Jaune (Challenge) → Rouge (Défaut). Registre public on-chain.

Solana ID Integration : score RociFi mis à jour après chaque verdict. Score <400 → futures ISAs blocké. Score>700 → bonus acceleration (cap +10%, durée -3mois).

Arcx/Spectra : metadonnées publiques, recovery après 24 mois sans incident.

Exécution & Slashing

EVM : Kleros Governor exécute via proxy (0xSplits + ERC-3525). Verdict Rouge → Governor appelle revoke() + burn(CPPS).

Solana : Alumni Council (Realms vote) ou Kleros pont (Wormhole). Exécution : Permanent Delegate RBK appelle burn(Token-2022) atomiquement.

Penalty : montant brûlé = cap restant * 1.2 (120% slashing pour dissuasion). Frais arbitrage non-remboursables.

Arbitrage Décentralisé

Tier 1 (Inclus) : Alumni Council (Realms) juge disputes <10k TND. Vote quadratique. Délai=10j.

Tier 2 (Kleros ERC-792) : Disputes >10k ou appel. Jurés internationaux, preuves on/off-chain. Frais rembour si apprenant gagnant. Délai=30j.

Tier 3 (TBD) : Recourse légale hors-chaîne en Tunisie (notaire + contrat local). RBK couvre frais si apprenant gagn. Fallback déf.
