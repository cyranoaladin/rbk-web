Philosophie

À l’Académie, un capstone est traité comme un produit de studio : threat model avant code, tests exhaustifs, CI verte et documentation d’exploitation.

Les 3 projets signatures

Capstone 1

Wallet & Reliability Pack

Focus : Frontend / UX / RPC Failover

User stories
  • Feedback précis (signing/confirming).
  • Retry auto (backoff).
  • Journal d’échecs.
Tech specs
  • Next.js + TanStack Query.
  • State machine stricte.
  • Failover RPC (primaire + fallback).
Checklist sécurité
  • Pas de clés privées stockées.
  • Inputs sanitizés.
  • Gestion du slippage.
Capstone 2

Tokenization & Admin Control

Focus : RBAC / Logique métier / DeFi

User stories
  • SuperAdmin : nommer/révoquer operator.
  • Operator : freeze compte suspect.
  • Auditeur : log on-chain des actions admin.
Tech specs
  • Solidity (AccessControl) / Anchor (PDA).
  • Invariants supply = somme des soldes.
  • Events pour toute mutation.
Checklist sécurité
  • Two-step transfer (admin).
  • Checks-Effects-Interactions.
  • Access control strict.
Capstone 3

Digital Assets & Utility

Focus : Performance / NFT / Gaming

User stories
  • Gating par NFT.
  • Rewards quotidiennes.
  • Mint optimisé.
Tech specs
  • Merkle tree ou signature verifier.
  • Compression (cNFT) ou ERC721A.
Checklist sécurité
  • Bot protection.
  • Anti-replay.
  • Metadata IPFS/Arweave.

Matrice d'évaluation

Pilier Pts Critères Fail condition
Ingénierie 40 Archi propre, modularité, gestion erreurs. Code monolithique, unwrap sauvages.
Sécurité 30 Threat model, validations, tests limites. Vulnérabilité critique.
Delivery 20 CI/CD, Git flow, reproductibilité. Build cassé, pas de doc.
Comms 10 Démo claire, pitch, justification des choix. Pitch brouillon, incapacité à expliquer.
