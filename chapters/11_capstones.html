Capstones (Projets Signatures)

Le standard "Studio". Pas de projets d'école, mais des produits "Mainnet-Ready" et "Audit-Ready".

Philosophie : Le Standard "Studio"

Chez RBK, un Capstone n'est pas un exercice académique. C'est un produit qui respecte les standards d'un studio professionnel. Il ne s'agit pas de prouver que "ça marche", mais que "ça ne peut pas casser".

Mindset

"Audit Ready"

Liste de Contrôle Studio-Grade (Non-Négociable)

Sécurité Threat Model formalisé AVANT le code Doc STRIDE
Qualité Zéro warning linter, coverage > 80% Rapport CI
Ops Déploiement scripté et reproductible Makefile

Les 3 Projets Signatures

Capstone 1

Wallet & Reliability Pack

Focus : Frontend / UX / RPC Failover

User Stories
  • Feedback précis (States: Signing, Confirming).
  • Retry Auto (Backoff exponentiel).
  • History Log des échecs.
Tech Specs
  • Next.js + TanStack Query.
  • State Machine stricte.
  • Failover RPC (Primaire + Fallback).
Liste de Contrôle Sécurité
  • Pas de Private Key stockée.
  • Sanitize inputs.
  • Gestion Slippage.
Capstone 2

Tokenization & Admin Control

Focus : RBAC / Logique Métier / DeFi

User Stories
  • SuperAdmin : Nommer/Révoquer Operator.
  • Operator : Freeze compte suspect.
  • Auditeur : Log on-chain actions admin.
Tech Specs
  • Solidity (AccessControl) / Anchor (PDA).
  • Invariant : TotalSupply == Sum Balances.
  • Events pour tout changement d'état.
Liste de Contrôle Sécurité
  • Two-Step Transfer (Admin).
  • Checks-Effects-Interactions.
  • Access Control strict.
Capstone 3

Digital Assets & Utility

Focus : Performance / NFT / Gaming

User Stories
  • Gating : Accès Beta si NFT Owner.
  • Claim : Reward 1x / jour.
  • Mint optimisé.
Tech Specs
  • Merkle Tree (Allowlist) ou Signature Verifier.
  • Compression (cNFT) ou ERC721A.
Liste de Contrôle Sécurité
  • Bot Protection (Nonce/Captcha).
  • Anti-Replay Attack.
  • Metadata IPFS (No HTTP).

Matrice d'Évaluation (Rubric)

Pilier Pts Critères Fail Condition
Ingénierie 40 Archi propre, Modularité, Gestion erreurs. Code monolithique, "Unwrap" sauvages.
Sécurité 30 Threat Model, Validation inputs, Tests limites. Vulnérabilité critique, Clé privée dans repo.
Delivery 20 CI/CD, Git Flow, README, Reproductible. Pas de doc, Build cassé.
Comms 10 Démo claire, Pitch deck, Justification choix. Démo illisible, Incapable de justifier.
