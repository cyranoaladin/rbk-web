Le rite de passage ultime. Nous construisons un Automated Market Maker (Constant Product) étape par étape. D'abord minimal (swap simple), puis complet (frais, tokens LP, sécurité).

Objectifs

  • Implémenter la courbe x * y = k.
  • Ajouter Fees, LP Tokens & Slippage Protection.

Phase 1 : AMM Minimal

Construction du moteur de base. Pas de cloches ni de sifflets. Juste la mathématique brute.

State & PDA Un compte Pool qui stocke les `bump` et `reserves`. Un PDA Authority.
Swap Instruction Input Amount connus. Calculer Output Amount. Transférer via CPI.
SWAP.RS
let k = pool.reserve_x * pool.reserve_y;

// User send X (new_x > old_x)
let new_reserve_x = pool.reserve_x + amount_in;

// Calculate Y to keep K const
let new_reserve_y = k / new_reserve_x;
let amount_out = pool.reserve_y - new_reserve_y;

// Execute Transfer
token::transfer(ctx, amount_out)?;
02

Phase 2 : Features Essentielles

Un AMM sans fees est un AMM mort. Un AMM sans Slippage Protection est un piège.

LP Tokens

Logic "Share of Pool". Mint/Burn synchronisé avec Add/Remove Liquidity.

Slippage

Paramètres min_out. Si amount_out < min_out → Revert.

Fees

Prélever 0.3% sur l'Input avant le calcul du Swap. Accumuler dans la Pool.
