11. Capstones : Projets Signatures (Mainnet-Ready)

Standards studio : produits auditables, sécurité prioritaire, livrables reproductibles.

CAPSTONES PROJECTS

La validation des compétences ne se fait pas par QCM, mais par la preuve de travail ("Proof of Work"). Chaque niveau se conclut par un livrable tangible et auditable.

Niveau 1

The Rust CLI Explorer

Développement d'un outil en ligne de commande (CLI) pur Rust, sans framework. L'outil doit se connecter au RPC Solana, parser manuellement les données binaires de comptes spécifiques et afficher un état structuré.

Rust Ownership Borsh Deserialization RPC Integration
Livrables attendus
  • Code source Rust (Crates.io standard)
  • Gestion d'erreurs (Result/Option)
  • Zéro `unwrap()` en production
  • Documentation technique (Cargo doc)
Niveau 2

Full Stack DeFi dApp

Conception et déploiement d'un protocole décentralisé complet (ex: Crowdfunding avec Escrow, ou DAO de vote). Le projet doit inclure le Smart Contract (Anchor), les tests unitaires et une interface utilisateur React connectée.

Anchor Framework CPI (Composability) PDA Management
Livrables attendus
  • Programme déployé (Devnet)
  • Tests TS (> 80% coverage)
  • Frontend fonctionnel (Wallet Adapter)
  • IDL public
Niveau 3

Rapport d'Audit Technique

Simulation d'une mission de "Red Teaming". L'étudiant doit analyser un protocole existant (ou fourni), identifier les vecteurs d'attaque potentiels et produire un rapport professionnel.

Exploit Writing Vulnerability Assessment Gas Optimization
Livrables attendus
  • Rapport PDF standard industriel
  • Classification (High/Med/Low)
  • PoC (Proof of Concept) des exploits
  • Recommandations de correctifs
