Performance: What the Builder Must Know

Compute Units (CU)

Every instruction has a "cost". Too many logs, too many CPIs, or bad crypto = Transaction Failed (ComputeBudgetExceeded).

Priority Fees

When the network is congested, paying more (Micro-lamports) lets you cut the line. It's a blockspace auction.

Logs Pollution

`msg!` is expensive (syscall). In prod, log the essential, not debug info.

Beware of simulated infinite loops Iterating over a large number of accounts in a single transaction is an anti-pattern on Solana. The CU limit will catch you. Prefer an "off-chain indexer + stateless logic" architecture or chunking.

Ops & Deliverables

Ops Best Practices

  • Deployments: Always check buffer authority before upgrade.
  • Rollback: Have a plan (e.g., revert to previous binary) if state migration fails.
  • Post-Mortem: If it breaks, write down why. No blame, but precision.

Exercise: Outage

"Your users complain 'the site is down'. Your RPC times out. The validator seems up. What do you do?"

1. Switch RPC (Failover).
2. Check Solana Beach / Solscan (Cluster status).
3. If Cluster is down, wait (and communicate).
