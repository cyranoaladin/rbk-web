02

CPI Discipline

1. Program ID Whitelist Never let user inject an arbitrary program. Check it is indeed TokenKegetc
2. Authority Check If transferring from an account, verify your program is the authority (PDA).
3. Signer Seeds invoke_signed with PDA seeds. Understand the under-the-hood mechanic.
CPI CHECKLIST
let cpi_program = ctx.accounts.token_program;

// 1. Check Program ID
if cpi_program.key() != &anchor_spl::token::ID {
  return Err(ProgramError::IncorrectProgramId.into());
}

// 2. Prepare Context
let cpi_accounts = Transfer {
  from: ctx.accounts.vault.to_account_info(),
  to: ctx.accounts.user_ata.to_account_info(),
  authority: ctx.accounts.vault_authority.to_account_info(),
};

// 3. Invoke Signed (PDA)
let seeds = &[&[b"vault", etc]];
token::transfer(
  CpiContext::new_with_signer(cpi_program, cpi_accounts, seeds),
  amount
)?;

Token-2022: Useful Extensions

A "Super-Set" of the Token program. Use only if justified.

Transfer Hooks

Exécuter une logique custom (whitelist, royalties) à chaque transfert. Révolutionnaire pour les enforces.

Confidential Transfers

Hide amounts (Zero Knowledge Proofs). Use cases: Payroll, Private B2B.

Metadata Interface

Embed metadata directly in the Mint (no need for external Metaplex account).
