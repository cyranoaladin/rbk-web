13. Capstones (Projets Signatures)

Le Standard "Studio" — 3 Projets "Mainnet-Ready" pour valider l'excellence (Audit Ready).

13.1 Philosophie : "Studio-Grade"

Un Capstone RBK n'est pas un projet d'école. C'est un produit qui respecte les standards d'un studio professionnel. Il ne s'agit pas de prouver que "ça marche", mais que "ça ne peut pas casser".

Sécurité : Threat Model J1
Qualité : 0 Warning, Coverage > 80%
Ops : Déploiement Scripté

13.2 Les 3 Projets Signatures

Capstone 1 : Wallet Reliability Pack

Frontend/UX Focus

Spécifications (User Stories)

  • Feedback : Spinner/Toast précis pour chaque état (Signing, Confirming).
  • Retry : Backoff exponentiel auto si RPC fail.
  • History : Log persistant des échecs pour debug.

Checklist Sécurité & Tech

  • Pas de Private Key stockée (Adapter Only).
  • Failover RPC (Config Primary/Fallback).
  • State Machine stricte (XState/TanStack).

Capstone 2 : Tokenization Admin Control

RBAC/DeFi Focus

Spécifications (User Stories)

  • Role : SuperAdmin peut nommer/révoquer Operator.
  • Freeze : Operator peut geler un compte (Compliance).
  • Audit : Historique on-chain immuable des actions Admin.

Checklist Sécurité & Tech

  • Access Control (Two-Step Transfer).
  • Pattern Anti-Reentrancy (Checks-Effects).
  • Events pour chaque changement d'état.

Capstone 3 : Digital Assets Ecosystem

NFT/Gaming Focus

Spécifications (User Stories)

  • Gating : Accès contenu uniquement avec NFT 'Early Bird'.
  • Claim : Reward une fois par jour (Logique temporelle).

Checklist Sécurité & Tech

  • Bot Protection (Captcha/Nonce).
  • Compression (cNFT) ou ERC721A.
  • Metadata IPFS (Pas de HTTP centralisé).
