« La Piscine n’est pas un cours, c’est un filtre. » — Critères d'Admission, Grilles de Scoring et Tests de Bypass.

H.1 Grille de Scoring « Piscine Rust »

Seuil d'Admission : 75/100
Score Final sur 100 points.
Poids Critère Indicateurs Observables
40% Aptitude Tech Progression exercices Rust, Qualité code final.
30% Résilience Rebond après échec, Constance de l'effort.
20% Collaboration Aide aux autres (Peer-learning).
10% Communication Clarté questionnement, Respect mentors.

H.1.1 Test d'Entrée Explorer (Bypass Genesis)

Pré-requis : Maîtrise prouvée de Rust ou C++ et des concepts Blockchain.

PARTIE 1 (45 min)

Théorie (QCM)

Account Model, Memory (Stack/Heap), Complexité Algorithmique.

PARTIE 2 (3 heures)

Pratique (CLI)

Parseur binaire + Signature Ed25519. KO si pas de tests unitaire.

PARTIE 3 (15 min)

Entretien Live

Code Review avec Lead Instructor. Justification mémoire.

Barème de Notation (Bypass Explorer)

Pts Critère Attendu Condition KO
40 Code Quality Rust idiomatique, Zero Clippy. unwrap() non géré.
30 Tests Unit tests couvrant edge cases. 0 TESTS FOURNIS.
30 Architecture Gestion Erreurs (Result), Structs. Code non structuré.

H.1.2 Test d'Entrée Bâtisseur (Bypass Explorer)

Pré-requis : Portfolio 2+ ans sur stack cible (Solana ou EVM).

1. Audit Readiness

Soumission d'un repo personnel existant. Vérification des critères "Studio" : CI/CD, Documentation, Tests E2E.

2. Exercice de Review

L'étudiant doit auditer une Pull Request (PR) contenant 3 vulnérabilités cachées (ex: Reentrancy, Overflow, Access Control).
