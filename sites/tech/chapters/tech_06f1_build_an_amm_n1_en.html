Build an AMM N1

Pool + LP invariants (vaults, deposits, withdrawals)

In N1 you create the pool state, the vaults, and the LP mint. The goal is to enforce ownership invariants and account identity so that no user can spoof the pool.

Pool state (minimum)

The pool must define immutable identity: which two mints are traded, which vaults hold reserves, and which LP mint represents liquidity shares.

Mints
  • mint_a and mint_b are fixed after init.
  • Decimals are known and used consistently in math.
Vaults
  • vault_a holds mint_a only.
  • vault_b holds mint_b only.
  • Vault owner is the pool PDA (not a user key).
LP Mint
  • lp_mint is fixed after init.
  • Mint authority is controlled by the program/PDA.

Non-negotiable invariants (N1)

Identity invariants
  • vault_a.mint == mint_a, vault_b.mint == mint_b.
  • vault_a.owner and vault_b.owner are the pool PDA.
  • lp_mint is the unique LP mint for this pool (no replacement).
Accounting invariants
  • Deposit increases reserves and mints LP according to defined formula.
  • Withdraw burns LP and decreases reserves consistently.
  • No negative balances, no silent overflow/underflow.
LP minting (minimum policy)
  • First deposit defines initial liquidity; document your policy explicitly.
  • Subsequent deposits mint LP proportional to contribution (consistent rounding policy).
  • Reject deposits that would mint zero LP (unless explicitly supported and tested).

Tests (minimum)

Negative tests
  • Vault spoof: user passes a token account they own as vault.
  • Mint mismatch: vault_a uses mint_b (must be rejected).
  • LP mint spoof: user passes a different lp_mint (must be rejected).
Regression tests
  • Init + deposit + withdraw cycles preserve invariants.
  • Boundary deposits: very small, very large, rounding edges.
  • Idempotence where expected (init fails on second attempt for same PDA).

Definition of Done (N1)

  • Pool init produces deterministic PDAs (pool, vaults) and fixed mints.
  • Deposit and withdraw enforce vault ownership and mint identity invariants.
  • At least 3 negative tests + 3 regression tests exist and are stable.
  • QA is green; routing works in English and French.
